# C04-09 递归调试方法

## 制作目标

- 学习目标：用 depth、进入日志和返回日志定位递归错误。
- 核心结论：让不可见的调用栈变成带缩进的文本轨迹，再核对参数、出口与返回值。
- 输出视频：`media/videos/recursion_chapter_scenes/1080p60/RecursionDebugVisualization.mp4`
- 预览图：`media/previews/recursion_debug_preview.png`

## 分镜与旁白

| 镜头 | 画面 | 旁白 |
| --- | --- | --- |
| 1 | `enter 3` 到 `enter 1` 按深度缩进。 | 进入函数时打印参数，缩进能直观看出当前调用层级。 |
| 2 | `leave 1` 到 `leave 3` 逆向出现。 | 子调用结束后再打印一次，就能看到控制权怎样回到父层。 |
| 3 | 左蓝右绿区分进入和返回。 | 颜色与缩进把压栈和弹栈两种阶段清楚分开。 |
| 4 | 出现参数、出口、返回值三项检查表。 | 先确认参数变小，再确认出口命中，最后检查子结果怎样合并。 |
| 5 | 二分和欧几里得参数示例闪现。 | 对区间递归打印 left、right、mid；对欧几里得打印每轮 a、b。 |

## 调试约定

```cpp
string indent(depth, '-');
cout << indent << "enter " << n << '\n';
trace(n - 1, depth + 1);
cout << indent << "leave " << n << '\n';
```

## 练习衔接

- `recursion-depth-trace`：复现缩进进入/返回轨迹。
- `recursion-binary-search-trace`：记录每层 mid 与最终下标。
- `recursion-euclid-trace`：观察余数如何推动参数走向出口。
