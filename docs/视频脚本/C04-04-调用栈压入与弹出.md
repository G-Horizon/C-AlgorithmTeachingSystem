# C04-04 调用栈压入与弹出

## 制作目标

- 学习目标：看懂栈帧、局部变量和返回值的流动顺序。
- 核心结论：调用按顺序压栈，最深层先返回，栈帧按后进先出弹出。

- 预览图：`media/previews/recursion_call_stack_preview.png`

## 分镜与旁白

| 镜头 | 画面 | 旁白 |
| --- | --- | --- |
| 1 | `sum(4)` 到 `sum(1)` 从下到上压入。 | 每次调用都会保存自己的参数、局部变量和返回位置。 |
| 2 | 上层栈帧暂停，最深层命中出口。 | 父调用并未消失，它在原位置等待子调用的答案。 |
| 3 | 右侧依次出现 1、3、6、10。 | 最深层先返回 1，结果再被上一层加上自己的 n。 |
| 4 | 栈帧从顶到底变绿并弹出。 | 调用栈遵循后进先出，因此展开与回收方向相反。 |
| 5 | 总结局部变量隔离。 | 每层虽然变量同名，但分别属于不同栈帧，不会互相覆盖。 |

## 代码映射

```cpp
long long child = recursiveSum(n - 1);
return n + child;
```

## 练习衔接

- `recursion-sum-stack`：读取并合并子调用返回值。
- `recursion-enter-leave`：直接输出压栈与弹栈顺序。
- `recursion-array-max`：将当前元素与子区间答案比较。
