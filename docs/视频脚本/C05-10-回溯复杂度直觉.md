# C05-10 回溯复杂度直觉

## 课时定位

- 所属章节：第五章 搜索与回溯。
- 核心目标：用分支因子 `b`、深度 `d` 和实际剪枝估算搜索规模。
- 建议时长：35 秒。
- 目标输出：`SearchComplexityVisualization.mp4`。

## 分镜安排

| 镜头 | 时长 | 画面 | 教学目的 |
| --- | --- | --- | --- |
| 1 | 4s | 标题，小树旁出现 `b` 和 `d`。 | 建立两个估算参数。 |

## 旁白脚本

1. “写搜索前，先问两个问题：每一层有多少个候选，一共要做多少层决策。”
2. “如果每层最多 b 个分支，深度是 d，叶子上界就是 b 的 d 次方。只要多几层，树冠就会迅速膨胀。”
3. “真正访问的还包括中间节点，总数是从一到 b 的 d 次方的累加，通常与最后一层是同一量级。”
4. “但不是每道题的分支数都不变。全排列第一层有 n 个选择，之后逐层减一，叶子数是 n 的阶乘。”
5. “剪枝会让实际树变小，但不要只靠乐观想象。先估最坏上界，再结合约束判断有效分支。”

## 代码画面映射

```cpp
if (leaves > limit / b) overBudget = true;

```

## 练习衔接

- `search-budget-check`：在乘法前检查是否超过搜索预算。
- `search-full-tree-node-count`：区分叶子数和整棵树的节点数。
