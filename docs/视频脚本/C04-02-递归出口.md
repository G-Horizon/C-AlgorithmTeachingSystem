# C04-02 递归出口

## 制作目标

- 学习目标：理解出口如何停止压栈并启动回收。
- 核心结论：出口要能直接回答最小问题，而且递归参数必须真正到达它。

- 预览图：`media/previews/recursion_base_case_preview.png`

## 分镜与旁白

| 镜头 | 画面 | 旁白 |
| --- | --- | --- |
| 1 | 左侧栈从 3 压到 -1 并变红。 | 没有出口时，每一层都会继续创建新调用，调用栈最终会被耗尽。 |
| 2 | 右侧只压入 3、2、1。 | 加入 `n==1` 的判断后，最小问题可以直接返回。 |
| 3 | 箭头从最深层逐层向上。 | 出口命中的瞬间，执行方向由继续压栈变成逐层返回。 |
| 4 | 对比“写了出口”和“可达出口”。 | 出口条件存在还不够，参数必须每次靠近它。 |
| 5 | 总结最小输入检查法。 | 写完递归，先用 n=0 或 n=1 单独检查边界。 |

## 代码映射

```cpp
if (n == 0) return;
printUp(n - 1);
cout << n;
```

## 练习衔接

- `recursion-print-up`：先递归再输出，观察出口后的回收顺序。
- `recursion-digit-sum`：数字缩小到 0 时返回。
- `recursion-gcd`：第二个参数为 0 时返回最大公约数。
