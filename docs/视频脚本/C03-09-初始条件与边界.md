# C03-09 初始条件与边界

## 1. 视频定位

- 章节：第三章 递推算法
- 课时：第 9 课
- 目标时长：35 秒左右
- 核心目标：让学生理解“递推式正确”不等于“程序一定正确”，初始条件、循环起点和数组边界共同决定递推能否启动。
- 前置知识：状态定义、一维递推、二维递推。
- 后续衔接：递归出口、动态规划初始化与遍历顺序。

## 2. 学习目标

看完本课后，学生应能：

1. 在写递推循环前先列出最小规模问题的答案。
2. 根据递推依赖确定循环起点，而不是机械地从 0 或 1 开始。
3. 发现 `f[i - 2]` 在 `i = 1` 时会访问不存在的 `f[-1]`。
4. 用“极小输入”检查 `n = 0`、`n = 1`、单行和单列等边界。
5. 按“定义状态 → 写初值 → 定方向和起点 → 递推 → 检查边界”的顺序完成程序。

## 3. 本课统一示例

使用爬楼梯状态：

```text
f[i] = 走到第 i 阶的方法数
```

允许每次走 1 阶或 2 阶，则：

```text
f[0] = 1
f[1] = 1
f[i] = f[i - 1] + f[i - 2]，i >= 2
```

这里把“站在第 0 阶”看作一种起点方案。这样 `n = 0` 有清晰答案，也能让 `f[2] = f[1] + f[0]` 自然成立。

## 4. 分镜安排

### 镜头一：公式看似正确，表格却启动不了

- 画面出现状态格 `f[0]` 到 `f[6]`。
- 只写出递推式 `f[i] = f[i-1] + f[i-2]`，但所有格子都是问号。
- 计算箭头试图从前两个问号推出 `f[2]`，随后整条状态链变红。
- 屏幕提示：`没有初值，递推没有已知起点`。

### 镜头二：错误循环访问到不存在的位置

- 展示错误循环 `for (int i = 1; i <= n; i++)`。
- 当 `i = 1` 时，两支依赖箭头分别指向 `f[0]` 和 `f[-1]`。
- `f[-1]` 显示在数组外，并以红色报警。
- 屏幕提示：`先看依赖，再决定循环从哪里开始`。

### 镜头三：补上初值，修正循环起点

- `f[0] = 1`、`f[1] = 1` 先变成绿色已知格。
- 循环起点从 `i = 1` 修正为 `i = 2`。
- `f[2]` 到 `f[6]` 依次被点亮，得到 `2、3、5、8、13`。
- 同步展示三步：`初值 → 起点 → 递推`。

### 镜头四：动画对象映射到 C++

- 左侧为正确状态表，右侧展示代码。
- 高亮 `vector<long long> f(n + 2, 0)`：数组为需要访问的下标留出空间。
- 高亮 `f[0] = 1; f[1] = 1;`：最小状态先成为已知。
- 高亮 `for (int i = 2; i <= n; i++)`：循环从第一个未知状态开始。

### 镜头五：用极小输入做边界体检

- 依次弹出 `n = 0 → 1`、`n = 1 → 1`、`n = 2 → 2`。
- 总结卡显示：
  1. 状态定义能否覆盖最小输入？
  2. 所有依赖下标是否存在？
  3. 循环是否从第一个未知状态开始？

## 5. 逐镜头字幕脚本

### 0—6 秒

> 递推式写对了，为什么程序还是算不出来？因为未知状态不能从另一堆未知状态里凭空出现。

### 6—12 秒

> 如果循环从一开始，`f[i-2]` 会在 `i=1` 时变成 `f[-1]`。先看依赖，才能确定循环起点。

### 12—21 秒

> 对爬楼梯来说，先把 `f[0]` 和 `f[1]` 设为 1，再从 `i=2` 开始，后面的状态才能一个接一个被推出。

### 21—29 秒

> 数组容量、初始条件和循环起点不是三个零散细节，它们共同保证每次读取的旧状态都已经存在并且算好。

### 29—35 秒

> 最后用 0、1、2 这样的极小输入体检边界。记住：先定义，再初始化，最后递推。

## 6. 动画对象到 C++ 代码映射

| 动画对象 | 含义 | C++ 对应 |
| --- | --- | --- |
| 状态格 `f[i]` | 走到第 i 阶的方法数 | `vector<long long> f` |
| 绿色初始格 | 已知的最小状态 | `f[0] = 1; f[1] = 1;` |
| 蓝色依赖箭头 | 当前状态读取的旧状态 | `f[i - 1]`、`f[i - 2]` |
| 红色数组外格 | 非法下标 | `f[-1]` |
| 逐格点亮 | 按正确方向推进递推 | `for (int i = 2; i <= n; i++)` |
| 极小输入卡 | 边界测试 | `n = 0, 1, 2` |

## 7. 课时示例代码

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    int n;
    cin >> n;

    vector<long long> f(n + 2, 0);
    f[0] = 1;
    f[1] = 1;

    for (int i = 2; i <= n; i++) {
        f[i] = f[i - 1] + f[i - 2];
    }

    cout << f[n];
    return 0;
}
```

## 8. 练习衔接

1. `边界版爬楼梯`：显式处理 `n = 0`，根据两项依赖确定循环起点。
2. `补齐递推状态表`：给定两个初值，输出从 `f[0]` 到 `f[n]` 的完整状态表。
3. `哨兵边界走方格`：用 `dp[0][1] = 1` 统一首行、首列与 `1 × 1` 网格的处理。

## 9. 制作检查

- 红色只用于错误初值、越界访问和失败状态。
- `f[-1]` 必须明确画在数组边界之外。
- 修复后必须先点亮初值，再点亮递推结果，避免把“初始化”和“计算”混在一起。
- 字幕中的状态含义、代码下标和练习题保持一致。
- 最终预览图应能同时看见绿色初值、蓝色递推链和边界检查主题。
