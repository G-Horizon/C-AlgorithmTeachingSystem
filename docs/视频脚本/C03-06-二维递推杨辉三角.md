# C03-06 二维递推：杨辉三角

## 课时定位

- 所属章节：第三章 递推算法
- 前置课时：C03-05 滚动变量优化
- 核心目标：让学生把一维“向右填表”的直觉迁移到二维表格，理解二维递推中的边界与上方依赖。
- 关键公式：`f[i][j] = f[i - 1][j - 1] + f[i - 1][j]`

## 旁白脚本

### 镜头 1：引入二维递推

画面：

- 中央出现标题“递推算法：二维递推”
- 副标题“把一张表从上往下填满”

旁白：

> 前面我们一直在一条线上做递推。现在把状态放进二维表格里，递推的方向就从“向右走”，变成“从上往下填”。

### 镜头 2：建立二维状态表

画面：

- 杨辉三角的空格子逐行出现
- 每个格子用 `f[i][j]` 标出行列位置

旁白：

> 杨辉三角里，`f[i][j]` 表示第 `i` 行第 `j` 个数。每一行都是一个小数组，所有行叠起来，就是一张二维状态表。

### 镜头 3：边界先确定

画面：

- 每一行最左和最右格依次高亮
- 边界格填入 `1`
- 底部提示“边界不用推，先放好”

旁白：

> 二维递推最容易漏的是边界。杨辉三角每一行的两端都是 `1`，这些格子不用公式推，先固定下来。

### 镜头 4：内部格来自上方两格

画面：

- 聚焦 `f[5][3]`
- 左上格和右上格被高亮
- 两条箭头指向当前格
- 公式卡片显示 `f[5][3] = f[4][2] + f[4][3] = 3 + 3 = 6`

旁白：

> 内部格就要看上一行。当前格的值，等于左上方和右上方两个旧状态相加。只要上一行已经算好，这一格就有了来源。

### 镜头 5：按行填表

画面：

- 第 4 行、第 5 行从左到右填入内部格
- 每一步短暂高亮两个来源和当前格
- 底部提示“先上一行，再下一行”

旁白：

> 所以循环顺序也很自然：先处理第 1 行，再处理第 2 行，一行一行向下。行内从左到右填，只要引用的都是上一行，就不会打乱依赖。

### 镜头 6：映射到代码

画面：

- 左侧保留小型杨辉三角
- 右侧显示代码：

```cpp
for (int i = 1; i <= n; i++) {
    f[i][1] = f[i][i] = 1;
    for (int j = 2; j < i; j++) {
        f[i][j] = f[i - 1][j - 1] + f[i - 1][j];
    }
}
```

旁白：

> 代码里，外层循环控制行，先放好这一行的两个边界。内层循环只处理内部格，公式正好对应左上和右上两个箭头。

### 镜头 7：总结检查表

画面：

- 标题“二维递推检查表”
- 三条检查项依次出现：
  1. 边界先放 `1`
  2. 内部看上一行两格
  3. 按行向下填表

旁白：

> 写二维递推题时，先问边界在哪里，再问当前格依赖谁，最后确定填表顺序。杨辉三角就是最清晰的入门模板。
