# C03-02 从已知推出未知

## 课时定位

- 所属章节：第三章 递推算法。
- 前置课时：C03-01 什么是状态。
- 核心问题：状态已经定义好了，为什么代码必须按某个顺序一格一格填表？
- 学习目标：让学生看见“旧状态先知道，新状态由旧状态推出”，理解递推方向、依赖箭头和循环起点。
- 预计视频长度：35 到 45 秒。

## 分镜安排规划

| 镜头 | 时长 | 画面安排 | 讲解重点 |

| 1 | 4s | 标题“递推算法：从已知推出未知”，副标题“先有旧格子，才有新格子”。 | 递推不是猜答案，而是从已经确定的信息向前推进。 |
| 2 | 6s | 一排状态格 `f[1]` 到 `f[6]` 出现，`f[1]=1`、`f[2]=2` 先发光，右侧格子显示 `?`。 | 递推开始前，需要有初始的已知状态。 |
| 3 | 7s | 两条箭头从 `f[1]`、`f[2]` 指向 `f[3]`，公式 `f[3] = f[2] + f[1]` 在上方出现。 | 新状态必须依赖已经算好的旧状态。 |
| 4 | 7s | `f[3]` 从 `?` 变为 `3`，背景变成绿色，已知区从前两个格子扩展到前三个格子。 | 算完的新格子，会立刻加入已知区。 |
| 5 | 10s | 指针移动到 `i=4`、`i=5`、`i=6`，每次箭头从前两个已知格汇入当前格，值依次变为 `5`、`8`、`13`。 | 递推的方向是从小下标到大下标，已知区不断向右生长。 |
| 6 | 6s | 右侧出现代码片段，循环行 `for (int i = 3; i <= n; ++i)` 高亮，表格继续显示已知区。 | 循环起点来自第一个未知格，循环顺序必须满足依赖关系。 |
| 7 | 4s | 总结三句话：“先初始化”“看依赖”“按方向推进”。 | 形成递推代码的基本检查清单。 |

## 视频脚本

### 镜头 1：建立主题

旁白：

> 上一节我们给每个位置起了状态名。现在的问题是：这些格子的值从哪里来？

画面：

- 深色背景。
- 标题居中出现：`递推算法：从已知推出未知`。
- 副标题出现：`先有旧格子，才有新格子`。

### 镜头 2：先亮起初始状态

旁白：

> 递推不是从空白中变出答案。它一定先有几个确定的起点，比如 `f[1]=1`、`f[2]=2`。

画面：

- 六个状态格从左到右排开：`f[1]`、`f[2]`、`f[3]`、`f[4]`、`f[5]`、`f[6]`。
- 前两个格子显示数值并变成绿色。
- 后四个格子显示问号。
- 表格下方出现“已知区”标签，覆盖前两个格子。

### 镜头 3：箭头指向第一个未知格

旁白：

> 如果当前格子依赖前两个格子，那么要算 `f[3]`，就必须先看 `f[2]` 和 `f[1]`。

画面：

- `f[3]` 被蓝色边框突出。
- 箭头从 `f[1]` 和 `f[2]` 指向 `f[3]`。
- 上方出现公式：`f[3] = f[2] + f[1]`。
- 数字 `2 + 1` 在公式下方短暂出现。

### 镜头 4：新状态加入已知区

旁白：

> 算出 `3` 之后，`f[3]` 就不再是未知格。它也能帮助后面的格子。

画面：

- `f[3]` 中的问号变成 `3`。
- `f[3]` 背景变成绿色。
- “已知区”横条向右扩展，覆盖前三个格子。

### 镜头 5：循环推进

旁白：

> 然后指针走到 `i=4`，继续用前两个已知格推出当前格。每算出一个，新格子就变成下一轮的旧状态。

画面：

- 指针依次移动到 `i=4`、`i=5`、`i=6`。
- 每一步显示：
  - 当前格高亮；
  - 前两个格子用箭头指向当前格；
  - 当前值由 `?` 变成具体数字；
  - 已知区向右扩展一格。
- 数值依次为 `5`、`8`、`13`。

### 镜头 6：映射到代码

旁白：

> 所以代码里的循环不是随便写的。`i=3` 是第一个未知格，`i++` 表示已知区从左往右推进。

画面：

- 表格缩到左侧。
- 右侧出现代码：

```cpp
f[1] = 1;
f[2] = 2;
for (int i = 3; i <= n; ++i) {
    f[i] = f[i - 1] + f[i - 2];
}
```

- 高亮 `for (int i = 3; i <= n; ++i)`。
- 再高亮 `f[i - 1] + f[i - 2]`。

### 镜头 7：总结

旁白：

> 写递推时，先初始化，再看当前格依赖谁，最后按照依赖允许的方向推进。

画面：

- 三个检查项依次出现：
  1. `先初始化`
  2. `看依赖`
  3. `按方向推进`

## 练习衔接

- 简单数列求值：给定 `a[1]`、`a[2]`，按照 `a[i]=a[i-1]+a[i-2]` 求第 `n` 项。
- 填表过程输出：输出每一次新算出的状态值，强化“已知区扩展”的直觉。
- 找循环起点：给定初始状态和转移式，判断循环应从哪个下标开始。
