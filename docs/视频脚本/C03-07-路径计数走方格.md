## 旁白脚本

### 镜头 1：引入路径计数

画面：

- 中央出现标题“递推算法：路径计数”
- 副标题“走到一个格子的方法数从哪里来”

旁白：

> 这一节我们继续做二维递推，不过表格变成了一张小地图。问题是：从左上角出发，只能向右或向下，走到右下角一共有多少种走法？

### 镜头 2：看见移动规则

画面：

- 4 x 5 网格出现
- 小机器人站在左上角
- 右下角出现目标旗
- 两个方向箭头标注“向右”和“向下”

旁白：

> 因为每一步只能向右或向下，所以机器人不会绕圈，也不会回头。这个限制很重要，它让我们可以从左上到右下，一格一格把答案推出来。

### 镜头 3：边界只有一种走法

画面：

- 网格中的每个格子变成 `dp[i][j]`
- 第一行依次填入 `1`
- 第一列依次填入 `1`
- 底部提示“边界不用选择，只有一路走到底”

旁白：

> 先看边界。第一行的格子只能一直向右走到，第一列的格子只能一直向下走到，所以它们的方法数都是 `1`。边界先设好，后面的格子才有来源。

### 镜头 4：内部格来自上方和左方

画面：

- 聚焦 `dp[3][4]`
- 上方格 `dp[2][4] = 4` 高亮
- 左方格 `dp[3][3] = 6` 高亮
- 两条箭头指向当前格
- 公式卡片显示 `dp[3][4] = dp[2][4] + dp[3][3] = 4 + 6 = 10`

旁白：

> 要走进一个内部格，最后一步只有两种可能：要么从上方往下走，要么从左方往右走。所以当前格的方法数，就是上方方法数加左方方法数。

### 镜头 5：按行填满整张表

画面：

- 从第二行第二列开始，按行填充表格
- 每一步短暂高亮“上方、左方、当前格”
- 底部提示“先算来源，再算当前”

旁白：

> 填表顺序也顺着这个依赖来。我们从左上往右下推进，保证每次计算当前格时，上方和左方都已经算好了。

### 镜头 6：映射到代码

画面：

- 左侧保留填好的 `4 x 5` 网格
- 右下角答案 `35` 放大
- 右侧显示代码：

```cpp
for (int i = 1; i <= n; i++) dp[i][1] = 1;
for (int j = 1; j <= m; j++) dp[1][j] = 1;

for (int i = 2; i <= n; i++) {
    for (int j = 2; j <= m; j++) {
        dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
    }
}
```

旁白：

> 代码里先初始化第一列和第一行，再用双重循环填内部格。最终右下角 `dp[n][m]`，就是走完整张地图的方法数。

### 镜头 7：总结检查表

画面：

- 标题“路径计数检查表”
- 三条检查项依次出现：
  1. 第一行、第一列先设 `1`
  2. 内部格看上方和左方
  3. 右下角就是答案

旁白：

> 做路径计数题时，先找边界，再找当前格从哪里来，最后确认答案位置。只要这三件事清楚，公式就不是背出来的，而是从地图里长出来的。
