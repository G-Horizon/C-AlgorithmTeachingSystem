# C03-08 数塔递推

## 课时定位

- 所属章节：第三章 递推算法
- 前置课时：C03-07 路径计数：走方格
- 核心目标：让学生理解二维递推中的“最值转移”：当前格的最优答案来自下面两个已知子问题中的较大者。
- 关键公式：`f[i][j] = a[i][j] + max(f[i + 1][j], f[i + 1][j + 1])`

## 旁白脚本

### 镜头 1：引入数塔递推

画面：

- 中央出现标题“递推算法：数塔递推”
- 副标题“从底层答案推回顶端”

旁白：

> 这一节我们看一个很有画面感的递推题：数塔。你从最上面的数字出发，每一层只能走到左下或右下，目标是让路径上的数字总和最大。

### 镜头 2：看懂路径规则

画面：

- 数字三角形出现：

```text
        7
      3   8
    8   1   0
  2   7   4   4
4   5   2   6   5
```

- 从顶部 `7` 画出通向下一层 `3` 和 `8` 的箭头
- 底部提示“每一步只能向左下或右下”

旁白：

> 每一步只有两个选择：左下，或者右下。看起来像是在找一条路，但如果我们从顶往下试，每个位置都可能被不同路径走到，信息会比较散。

### 镜头 3：定义状态，并从底层开始

画面：

- 右侧出现状态卡片：`f[i][j] = 从 a[i][j] 走到底的最大和`
- 底层 `4, 5, 2, 6, 5` 依次高亮，直接填成已知状态
- 提示“站在底层时，已经没有下一步”

旁白：

> 换一个方向想会更顺。定义 `f[i][j]` 表示：从这个数字出发，一直走到底，能得到的最大和。站在最底层时已经没有下一步，所以底层的 `f` 就等于它自己。

### 镜头 4：当前格依赖下面两个格

画面：

- 聚焦第 4 行第 2 个数 `7`
- 下方两个状态 `5` 和 `2` 发光
- 公式卡片显示：`f[4][2] = 7 + max(5, 2) = 12`
- `7` 所在格更新为 `12`

旁白：

> 现在看这个 `7`。从它往下只能走到 `5` 或 `2`。如果后面的最大和已经知道，那当前格只要加上两个选择里更大的那个：`7 + max(5, 2)`，得到 `12`。

### 镜头 5：自底向上填表

画面：

- 从倒数第二行开始向上填：
  - 第 4 行变成 `7, 12, 10, 10`
  - 第 3 行变成 `20, 13, 10`
  - 第 2 行变成 `23, 21`
  - 第 1 行变成 `30`
- 每一步都短暂显示“当前值 + max(左下, 右下)”

旁白：

> 数塔递推的方向是从下往上。因为要计算当前格，下面两个格必须先知道。每填完一层，上面一层就有了可靠来源，最后所有信息汇到顶端。

### 镜头 6：映射到代码

画面：

- 左侧保留填好的 `f` 数塔
- 顶端 `30` 用绿色描边放大
- 右侧显示代码：

```cpp
for (int j = 1; j <= n; j++) {
    f[n][j] = a[n][j];
}

for (int i = n - 1; i >= 1; i--) {
    for (int j = 1; j <= i; j++) {
        f[i][j] = a[i][j] + max(f[i + 1][j], f[i + 1][j + 1]);
    }
}
```

旁白：

> 代码也照着这个方向写。先初始化最底层，再让 `i` 从 `n - 1` 倒着走到 `1`。内层枚举这一行的每个位置，答案最后就在 `f[1][1]`。

### 镜头 7：总结检查表

画面：

- 标题“数塔递推检查表”
- 三条检查项依次出现：
  1. `f[i][j]` 表示从当前格走到底的最大和
  2. 底层直接初始化
  3. 当前格看左下和右下，倒着往上填

旁白：

> 数塔题的关键不是贪心地每一步选最大数字，而是让每个格子先知道“从我这里往下的最好结果”。这就是最值递推的味道。
