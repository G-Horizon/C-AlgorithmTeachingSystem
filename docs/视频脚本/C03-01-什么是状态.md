# C03-01 什么是状态

## 视频定位

- 所属章节：第三章 递推算法
- 学习目标：让学生把“状态”理解成对问题进度的记录，而不是抽象名词。
- 视觉核心：一条楼梯变成一排可填写的状态格，每个格子都回答“到这里有多少种方法”。
- 输出视频：`media/videos/state_definition_scene/480p15/RecurrenceStateVisualization.mp4`
- 预览图：`media/previews/recurrence_state_preview.png`

## 视频脚本

### 镜头 1：章节入口

旁白：

> 递推算法不是从公式开始的。它的第一件事，是决定我们要记录什么。

画面：

- 标题进入：`递推算法：什么是状态`
- 副标题进入：`先记录进度，再推出答案`

### 镜头 2：原问题太大

旁白：

> 比如这个问题：走到第 n 阶有多少种方法？如果只盯着 n，我们看到的是一个很大的终点。

画面：

- 中央问题卡片：`走到第 n 阶有多少种方法？`
- 右侧浮动 `n = ?`
- 卡片边缘用较暗颜色表现“还没有抓手”

### 镜头 3：把终点拆成沿途位置

旁白：

> 递推会把终点拆成一格一格的进度：第 0 阶、第 1 阶、第 2 阶……直到第 i 阶。

画面：

- 楼梯从左到右逐级出现
- 每一级底部标注 `0`、`1`、`2`、`3`、`4`、`5`、`i`

### 镜头 4：给每个位置一个状态名

旁白：

> 每个位置都需要一个记录盒子。我们把它叫做 `f[i]`。

画面：

- 楼梯上方出现状态格：`f[0]`、`f[1]`、`f[2]`、`f[3]`、`...`、`f[i]`
- 状态格与台阶用短箭头连接

### 镜头 5：把状态读成一句话

旁白：

> 但 `f[i]` 本身没有意义。真正重要的是这句话：`f[i]` 表示走到第 i 阶的方法数。

画面：

- 右侧定义卡片：`f[i] = 到第 i 阶的方法数`
- `i`、`第 i 阶`、`方法数` 三处依次高亮

### 镜头 6：从问全部到问每个位置

旁白：

> 原来我们只问“到终点有多少种”。现在我们改成问：到每个位置分别有多少种。

画面：

- 大问题卡片变成三个小问题卡片：`到 0 阶？`、`到 1 阶？`、`到 2 阶？`
- 小问题合并成状态表

### 镜头 7：先定义，再递推

旁白：

> 定义清楚以后，才轮到初始化和转移。比如 `f[0]=1`、`f[1]=1`，后面的格子下一节再用已知推出未知。

画面：

- `f[0]` 写入 `1`
- `f[1]` 写入 `1`
- 后续格子显示 `?`
- 底部提示：`本节重点：先说清 f[i] 记录什么`

### 镜头 8：代码映射

旁白：

> 写代码时，最值得先写下来的不是循环，而是状态含义。注释越清楚，后面的递推越不容易乱。

画面：

```cpp
// f[i] 表示走到第 i 阶的方法数
long long f[N];
f[0] = 1;
f[1] = 1;
// 答案在 f[n]
```

### 镜头 9：总结

旁白：

> 遇到递推题，先问三件事：它记录什么？下标表示什么？答案在哪个格子？

画面：

- 总结卡片：
  - `1. 它记录什么？`
  - `2. 下标表示什么？`
  - `3. 答案在哪个格子？`

## 练习衔接

- 入门：爬楼梯方法数。
- 基础：输出 `f[0]` 到 `f[n]` 的状态表。
- 进阶：2 x n 骨牌覆盖，练习把不同题目翻译成同一种状态表。
