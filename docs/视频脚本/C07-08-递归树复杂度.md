# C07-08 递归树复杂度

## 制作定位

- 核心目标：用“树高 × 每层工作量”估算典型分治复杂度，并识别快排退化。
- 建议成片：65～80 秒。

## 分镜

| 镜头 | 时长 | 画面 | 教学意图 |
| --- | --- | --- | --- |
| 1 | 8s | 规模 n 的根拆成两个 n/2，再拆成四个 n/4。 | 建立递归树。 |
| 2 | 10s | 沿一条根叶路径标注 `n→n/2→...→1`。 | 得到 log n 层。 |
| 3 | 12s | 每一层所有区间长度横向求和，均显示 n。 | 得到每层线性工作。 |
| 4 | 10s | 竖向层数与横向工作量相乘，出现 `n log n`。 | 合成复杂度。 |
| 5 | 12s | 二分查找只保留一个子节点，横向工作标为常数。 | 对比 O(log n)。 |
| 6 | 12s | 快排树从平衡形变成链形，高度由 log n 变 n。 | 解释最坏 O(n²)。 |

## 旁白提纲

1. “分析分治，先看一条路径有多深，再看同一层总共做多少工作。”
2. “规模每次减半，树高是对数级；归并每层处理全部 n 个元素。”
3. “二分只走一个分支，所以每层工作是常数。”
4. “分区若长期不平衡，快排递归树会退化成链。”

## 练习衔接与验收

- `divide-level-work`、`divide-full-tree-nodes`。
- 验收：层数口径要明确根为第 0 层；不得把所有二分递归都直接写成 O(n log n)。
