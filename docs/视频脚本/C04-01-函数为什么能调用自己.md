# C04-01 函数为什么能调用自己

## 制作目标

- 学习目标：把递归调用理解为“创建一份参数不同的新任务”，而不是函数在原地循环。
- 核心结论：递归需要相似子问题、不断缩小的规模和可直接处理的最小任务。

## 分镜与旁白

| 镜头 | 画面 | 旁白 |
| --- | --- | --- |
| 1 | 标题与副标题出现。 | 函数调用自己听起来像绕圈，其实每次调用都会创建一份参数不同的新任务。 |
| 2 | `solve(4)` 依次生成 `solve(3)`、`solve(2)`、`solve(1)`。 | 每张任务卡都只负责当前规模，再把更小的同类任务交给下一层。 |
| 3 | 四张卡片下方标出规模，数字持续减小。 | 规模必须严格变小。若仍调用 `solve(n)`，任务就会停在原地。 |
| 4 | `solve(1)` 变绿并标记“最小任务”。 | 到达最小问题后可以直接处理，不再创建新调用。 |
| 5 | 显示 C++ 递归骨架与总结式。 | 判断递归是否成立，就检查：子问题是否同类、规模是否变小、最小任务是否明确。 |

## 代码映射

```cpp
void solve(int n) {
    if (n == 1) return;
    solve(n - 1);
}
```

## 练习衔接

- `recursion-countdown`：当前层输出 n，下一层处理 n-1。
- `recursion-sum-to-n`：把答案拆成 n 与更小区间之和。
- `recursion-power-two`：用同构子问题计算 2 的幂。
