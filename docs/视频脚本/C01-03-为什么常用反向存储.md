# C01-03 为什么常用反向存储

## 课程定位

本视频解释高精度加减乘中最常见的存储习惯：把个位放在下标 0。学生已经知道如何把字符串拆成数组，本课要让他们看到“竖式从个位开始”和“数组下标从 0 开始”天然契合。

## 分镜安排

| 镜头 | 画面 | 讲解重点 |
| --- | --- | --- |
| 1 | 标题出现：为什么常用反向存储 | 存储方向不是规定，而是为了让计算更顺手 |
| 2 | 竖式 `987 + 65` 出现，个位列闪烁 | 加减乘都从个位开始 |
| 3 | 正向数组 `[9,8,7]` 出现，个位在最右边 | 正向存储时，下标 0 不是个位 |
| 4 | 数字翻转成 `[7,8,9]`，个位来到 `a[0]` | 反向存储让第 `i` 位和下标 `i` 对齐 |
| 5 | 两个反向数组上下对齐，`i=0,1,2` 依次扫过 | 缺位补 0，逐位计算更自然 |
| 6 | 代码区显示倒序读入循环和倒序输出 | 读入反着存，输出再反着打印 |

## 讲解脚本

开场：  
“如果我们已经会把字符串拆成数组，下一步要决定：数字从左到右存，还是从右到左存？”

竖式直觉：  
“看手算加法，第一步永远从个位开始。个位算完，进位给十位；十位算完，再给百位。”

正向存储的问题：  
“如果 `987` 正向存成 `[9,8,7]`，个位 7 在最右边。想从个位开始，就要不断用 `size - 1 - i` 这样的下标，代码会变绕。”

反向存储：  
“把它反过来存成 `[7,8,9]` 后，个位就在 `a[0]`，十位在 `a[1]`，百位在 `a[2]`。循环变量 `i` 自然对应第 `i` 个数位。”

输出提醒：  
“反向存储只是内部表示，输出答案时要从数组最后一位倒着打印回来。”

收束：  
“反向存储的核心好处是：从个位开始的竖式计算，正好从下标 0 开始写循环。”
