# C03-03 一维递推：爬楼梯

## 课时定位

- 所属章节：第三章 递推算法。
- 前置课时：C03-01 什么是状态；C03-02 从已知推出未知。
- 核心问题：为什么爬楼梯的当前状态等于前两个状态之和？
- 学习目标：让学生理解“最后一步分类”是递推公式的来源，掌握 `f[i] = f[i-1] + f[i-2]` 的状态含义、初始化、循环转移和答案输出。
- 预计视频长度：40 到 55 秒。

## 视频脚本

### 镜头 1：建立问题

旁白：

> 一次可以走 1 阶或 2 阶。要问走到第 n 阶有多少种方法，我们先看任意一个第 i 阶。

画面：

- 标题居中出现：`递推算法：一维递推爬楼梯`。
- 副标题出现：`答案藏在最后一步里`。

### 镜头 2：状态贴到楼梯上

旁白：

> 状态仍然这样定义：`f[i]` 表示走到第 i 阶的方法数。

画面：

- 楼梯第 `0` 到第 `i` 阶从左到右出现。
- 第 `i` 阶上方出现状态卡 `f[i]`。
- 下方文字提示：`状态：f[i] = 到第 i 阶的方法数`。

### 镜头 3：最后一步分类

旁白：

> 想到达第 i 阶，最后一步只有两种可能：从第 i-1 阶走 1 阶，或者从第 i-2 阶走 2 阶。

画面：

- 第 `i-1` 阶发光，黄色箭头跨 1 阶到第 `i` 阶，标签为 `走 1 阶`。
- 第 `i-2` 阶发光，蓝色箭头跨 2 阶到第 `i` 阶，标签为 `走 2 阶`。
- 两条箭头同时汇入第 `i` 阶。

### 镜头 4：推出公式

旁白：

> 这两类方案的最后一步不同，不会重复。于是到第 i 阶的方法数，就是这两部分相加。

画面：

- 左侧来源卡：`来自 i-1：f[i-1]`。
- 右侧来源卡：`来自 i-2：f[i-2]`。
- 两张卡片汇入中间公式：

```cpp
f[i] = f[i - 1] + f[i - 2]
```

- `+` 号短暂放大，强调数量合并。

### 镜头 5：逐格填表

旁白：

> 有了公式以后，先放好初始格：`f[0]=1`、`f[1]=1`。然后从 `i=2` 开始，每一格都由前两格推出。

画面：

- 表格出现：`f[0]` 到 `f[6]`。
- 初始格 `f[0]=1`、`f[1]=1` 变绿。
- 指针依次指向 `i=2`、`i=3`、`i=4`、`i=5`、`i=6`。
- 每一步都有两条依赖箭头从前两格指向当前格，当前值依次变为 `2,3,5,8,13`。

### 镜头 6：映射到代码

旁白：

> 代码就照着这件事写：先初始化，再从第一个未知格开始循环，最后输出 `f[n]`。

画面：

- 左侧保留已填好的表格。
- 右侧出现 C++ 代码：

```cpp
f[0] = 1;
f[1] = 1;
for (int i = 2; i <= n; ++i) {
    f[i] = f[i - 1] + f[i - 2];
}
cout << f[n];
```

- 依次高亮初始化、循环起点、转移式、答案格。

### 镜头 7：总结

旁白：

> 一维递推题常常就是这样：先用最后一步分类找到依赖，再按顺序填表。

画面：

- 三个检查项出现：
  1. `最后一步分类`
  2. `旧状态相加`
  3. `答案是 f[n]`

## 练习衔接

- 爬楼梯方法数：直接实现 `f[i]=f[i-1]+f[i-2]`。
- 输出爬楼梯状态表：观察每一格如何由前两格推出。
- 最后一步来源统计：输出每个 `i` 的两部分来源值，强化“分类相加”的理解。
