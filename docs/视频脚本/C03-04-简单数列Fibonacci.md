# C03-04 简单数列：Fibonacci

## 课时定位

- 所属章节：第三章 递推算法。
- 前置课时：C03-01 什么是状态；C03-02 从已知推出未知；C03-03 一维递推：爬楼梯。
- 核心问题：为什么 Fibonacci 的递推式固定是前两项相加，但代码里的起始下标和初始化会随题目变化？
- 学习目标：让学生理解递推数列的“前两项推出当前项”，掌握 `0-based` 与 `1-based` 两种常见定义方式，能根据题目定义选择初始化、循环起点和答案格。
- 预计视频长度：35 到 50 秒。

## 视频脚本

### 镜头 1：建立问题

旁白：

> Fibonacci 很容易背成一句话：当前项等于前两项之和。但真正写代码时，最容易错的不是公式，而是下标。

画面：

- 标题居中出现：`递推算法：简单数列 Fibonacci`。
- 副标题出现：`公式相同，下标先看题目`。

### 镜头 2：前两项推出当前项

旁白：

> 先看数值本身：`0, 1, 1, 2, 3, 5, 8`。从第三个数开始，每一项都由左边相邻的两项相加得到。

画面：

- 数列格子依次出现，格子上方写 `数值`，下方写位置标签。
- `1 + 1 = 2`、`1 + 2 = 3`、`2 + 3 = 5` 依次被箭头演示。
- 当前格短暂高亮，前两格用两条颜色不同的箭头指向它。

### 镜头 3：并排对比两种下标

旁白：

> 如果题目定义 `F(0)=0, F(1)=1`，那第一行从 `f[0]` 开始。另一些题会定义 `F(1)=1, F(2)=1`，这时第一项的下标就是 1。

画面：

- 上方一行：`0-based`，格子为 `f[0]=0, f[1]=1, f[2]=1, f[3]=2, f[4]=3, f[5]=5, f[6]=8`。
- 下方一行：`1-based`，格子为 `F[1]=1, F[2]=1, F[3]=2, F[4]=3, F[5]=5, F[6]=8`。
- 两行之间出现提示：`数值很像，下标不同`。

### 镜头 4：映射到代码

旁白：

> 下标一变，初始化和循环起点也会跟着变。`0-based` 从 `i=2` 算第一个未知格；`1-based` 已知前两格，所以从 `i=3` 开始。

画面：

- 左侧显示 `0-based` 代码：

```cpp
f[0] = 0;
f[1] = 1;
for (int i = 2; i <= n; i++) {
    f[i] = f[i - 1] + f[i - 2];
}
```

- 右侧显示 `1-based` 代码：

```cpp
f[1] = 1;
f[2] = 1;
for (int i = 3; i <= n; i++) {
    f[i] = f[i - 1] + f[i - 2];
}
```

- 高亮两组初始化，再高亮 `i = 2` 与 `i = 3`。

### 镜头 5：题目定义决定答案格

旁白：

> 所以看到 Fibonacci 题，不要先默写代码。先问：题目从 0 开始数，还是从 1 开始数？把初始格放对，再输出题目要求的 `f[n]`。

画面：

- 左侧题目卡：`给定 n，F(0)=0, F(1)=1，求 F(n)`，箭头指向 `0-based` 表。
- 右侧题目卡：`给定 n，F(1)=1, F(2)=1，求 F(n)`，箭头指向 `1-based` 表。
- 两张卡片下方分别出现答案标签：`答案：f[n]`。

### 镜头 6：总结

旁白：

> 简单数列递推可以按三步检查：先看题目下标，放好初始格，再从第一个未知格循环。

画面：

- 三个检查项出现：
  1. `先看下标`
  2. `放好初始格`
  3. `从第一个未知格循环`

## 练习衔接

- 0-based Fibonacci：按 `F(0)=0, F(1)=1` 输出第 `n` 项。
- 1-based Fibonacci：按 `F(1)=1, F(2)=1` 输出第 `n` 项。
- 下标对照表：输出前若干项的 `0-based` 与 `1-based` 对照，强化“题目定义决定初始化”。
