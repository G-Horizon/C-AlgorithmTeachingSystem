# C01-01 普通整数为什么不够用

## 课程定位

本视频是第一章高精度计算的入口课。学生需要先明白：`int`、`long long` 不是“更聪明的数字”，它们只是容量有限的容器；当题目数字超过容器范围时，必须改用字符串读入，并把数字拆成一位一位处理。

## 分镜安排

| 镜头 | 画面 | 讲解重点 |
| --- | --- | --- |
| 1 | 标题出现：普通整数为什么不够用 | 高精度不是炫技，而是为了解决普通整型装不下的问题 |
| 2 | `int`、`long long` 两个容量容器并排出现 | 变量类型都有上限，超过上限会溢出 |
| 3 | 输入 `100000000000000000000` 压向 `long long` 容器，出现红色溢出提醒 | 不能把超大整数直接放进整型变量 |
| 4 | 数字切换成一节节车厢，每个车厢只保存一位数字 | 大整数可以先作为 `string` 保存，再逐位拆开 |
| 5 | 代码区显示 `string s; cin >> s;` 和逐字符处理 | 读入方式先变，后续算法才有空间模拟竖式 |
| 6 | 总结三句：普通整型有范围、字符串能完整读入、逐位处理才是高精度入口 | 建立第一章的整体心智模型 |

## 讲解脚本

开场：  
“我们平时写 `int x` 或 `long long x`，看起来像是在声明一个数字。其实更准确地说，我们是在准备一个容量有限的盒子。”

容量对比：  
“`int` 大约只能装到二十多亿，`long long` 虽然大很多，但也不是无限大。当输入变成二十位、五十位甚至一千位时，它们都会装不下。”

溢出画面：  
“如果题目给出 `100000000000000000000`，这已经超过 `long long` 的范围。程序不会自动帮我们变聪明，直接用整型读入就可能得到错误结果。”

字符串车厢：  
“所以高精度的第一步，是把大整数当成一串字符读进来。字符串不会计算它的大小，它只是完整保存每一个字符。”

代码映射：  
“有了 `string s`，我们就可以从左到右扫描每一位，或者在后面的加减乘除里从右到左模拟竖式。真正的高精度计算，就是把一个巨大数字拆成很多个小数字来处理。”

收束：  
“这节课只记住一句话：普通整型装不下时，先用字符串保住完整信息，再按位处理。”
