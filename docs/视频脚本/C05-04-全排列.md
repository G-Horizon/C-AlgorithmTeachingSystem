# C05-04 全排列

## 课时定位

- 所属章节：第五章 搜索与回溯。
- 核心目标：用 `used` 数组保证每个数在一个排列中恰好使用一次。
- 建议时长：36 秒。
- 目标输出：`SearchPermutationVisualization.mp4`。

## 分镜安排

| 镜头 | 时长 | 画面 | 教学目的 |
| --- | --- | --- | --- |
| 1 | 4s | 三个路径槽与候选卡 1、2、3。 | 建立全排列任务。 |
| 2 | 7s | 1 进入第一槽，候选卡 1 变灰并标记 `used=true`。 | 说明不可重复使用。 |

| 5 | 7s | 六个排列列成两列，右侧显示 `n!`。 | 建立规模直觉。 |

## 旁白脚本

1. “全排列要把一到 n 的每个数都放进路径，而且每个数只能用一次。”
2. “选中数字一时，它进入 path，对应的 `used[1]` 变成 true。后续层再看到一，就直接跳过。”
3. “路径长度等于 n 时，所有槽位都填好了，当前 path 就是一个完整排列。”
4. “回退时不只要从 path 取出数字，还要把它的 used 恢复为 false，让兄弟分支能再次选它。”
5. “候选从小到大尝试，叶子就自然按字典序出现。n 每增加一，排列数会乘上新的 n。”

## 代码画面映射

```cpp
if (used[value]) continue;
used[value] = true; path.push_back(value);
dfs();
path.pop_back(); used[value] = false;
```
