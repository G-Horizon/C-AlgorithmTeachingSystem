# C03-05 滚动变量优化

## 课时定位

- 所属章节：第三章 递推算法。
- 前置课时：C03-01 什么是状态；C03-02 从已知推出未知；C03-03 一维递推：爬楼梯；C03-04 简单数列：Fibonacci。
- 核心问题：当递推只依赖最近几个旧状态时，为什么不一定要保存整张数组表？
- 学习目标：让学生理解“只保留后面还会用到的状态”，掌握用 `a`、`b`、`c` 三个变量完成 Fibonacci / 爬楼梯类递推的空间优化。
- 预计视频长度：30 到 45 秒。

## 视频脚本

### 镜头 1：建立问题

旁白：

> 前面我们用数组一格一格填表。可是有些递推，表虽然很长，真正计算下一格时，只会回头看最近的几个格子。

画面：

- 标题居中出现：`递推算法：滚动变量优化`。
- 副标题出现：`表可以很长，记忆可以很短`。

### 镜头 2：观察依赖范围

旁白：

> 以 Fibonacci 为例，算 `f[i]` 时只需要 `f[i-1]` 和 `f[i-2]`。更早的格子已经不会直接参与下一步计算。

画面：

- 横向状态表显示 `f[0]` 到 `f[8]`。
- 镜头依次高亮 `f[i-2]`、`f[i-1]` 和当前 `f[i]`。
- 公式卡显示：`f[i] = f[i-1] + f[i-2]`。
- 早期格子逐渐变暗，强调依赖窗口只有两个旧状态。

### 镜头 3：数组压缩成两个旧状态盒

旁白：

> 所以我们不用把整张表一直带在身上。只要用 `a` 记住较旧的那个状态，用 `b` 记住较新的那个状态，就足够推出下一项。

画面：

- 长数组变淡后缩到背景。
- 中央出现两个盒子：
  - `a = f[i-2]`
  - `b = f[i-1]`
- 下方提示：`只保留下一步还会用到的旧状态`。

### 镜头 4：先算 c，再滚动

旁白：

> 每一步先算新值：`c = a + b`。算完以后，窗口整体向前挪：原来的 `b` 变成新的 `a`，新算出的 `c` 变成新的 `b`。

画面：

- 三个盒子横向排列：`a`、`b`、`c`。
- 先显示 `c = a + b`，`c` 盒子亮起。
- 两条箭头依次出现：
  - `a = b`
  - `b = c`
- 用 2 到 3 轮数值更新演示窗口滚动：`0,1 -> 1`，`1,1 -> 2`，`1,2 -> 3`。

### 镜头 5：映射到代码

旁白：

> 写成代码时，关键就是这三行。注意不能先改 `a` 或 `b` 再去算 `c`，否则旧状态会被覆盖。

画面：

```cpp
long long a = 0, b = 1;
for (int i = 2; i <= n; i++) {
    long long c = a + b;
    a = b;
    b = c;
}
cout << b;
```

- 高亮 `long long c = a + b;`
- 再高亮 `a = b;` 和 `b = c;`
- 旁边显示空间变化：`数组 O(n)` 到 `变量 O(1)`。

### 镜头 6：总结

旁白：

> 滚动变量优化可以按三步判断：只保留后面还会用到的状态，先算新值，再把窗口整体向前滚动。

画面：

- 三个检查项出现：
  1. `只保留会再用的状态`
  2. `先算新值 c`
  3. `再整体滚动`

## 练习衔接

- 滚动变量求 Fibonacci：不用数组，只用变量输出 `F(n)`。
- 滚动变量求爬楼梯：把上一课的数组表压缩成两个变量。
- 输出滚动过程：按 `i:a+b=c` 打印每一步，检查更新顺序是否正确。
