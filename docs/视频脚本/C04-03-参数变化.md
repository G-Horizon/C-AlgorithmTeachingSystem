# C04-03 参数变化

## 制作目标

- 学习目标：能用“规模度量”检查递归是否走向出口。
- 核心结论：每次调用都要让 n、下标或区间长度严格变化。
- 输出视频：`media/videos/recursion_chapter_scenes/1080p60/RecursionParameterVisualization.mp4`
- 预览图：`media/previews/recursion_parameter_preview.png`

## 分镜与旁白

| 镜头 | 画面 | 旁白 |
| --- | --- | --- |
| 1 | 参数卡片 5、4、3、2、1 横向排开。 | 把参数看成递归的进度条，每一层都要更靠近出口。 |
| 2 | 数字间出现向右箭头。 | `n-1` 让规模递减；`index+1` 则让下标向终点推进。 |
| 3 | `solve(n)` 红色卡片停在原处。 | 参数不变会让同一个任务无限重复。 |
| 4 | `solve(n-1)` 绿色卡片指向出口。 | 好的递归调用应当能证明规模严格变小。 |
| 5 | 展示“手写前三层参数”。 | 调试时先写出前三层参数，通常立刻就能发现方向和边界错误。 |

## 代码映射

```cpp
void printRange(int current, int right) {
    if (current > right) return;
    cout << current << ' ';
    printRange(current + 1, right);
}
```

## 练习衔接

- `recursion-print-range`：单参数向右推进。
- `recursion-reverse-string`：下标前进、字符在回收阶段输出。
- `recursion-palindrome`：左右边界同时向中间收缩。
