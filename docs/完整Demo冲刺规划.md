# 完整 Demo 冲刺规划

## 目标

把当前项目从“课程内容与 OJ 原型”推进到“可完整演示的教学系统 demo”。演示时需要形成清晰闭环：

1. 学生进入课程，观看视频，阅读要点。
2. 学生进入练习，编写 C++，提交判题。
3. 学生看到测试点反馈、历史提交和继续学习入口。
4. 教师打开统计看板，看到练习热度、通过情况和薄弱题目。
5. 团队能说明后续如何根据学生反馈迭代课程、题目和讲解。

## 演示流程建议

1. 运行演示数据：

```powershell
python scripts/seed_demo_data.py
```

2. 作为学生启动完整 demo：

```powershell
powershell.exe -ExecutionPolicy Bypass -File scripts/student_start.ps1
```

也可以直接双击根目录 `start_student_demo.cmd`。

3. 启动带教师看板演示数据的完整 demo：

```powershell
powershell.exe -ExecutionPolicy Bypass -File scripts/demo_start.ps1 -SeedData
```

4. 打开：

```text
http://127.0.0.1:5173
http://127.0.0.1:5173/chapters/big-integer/lessons/big-integer-overflow
http://127.0.0.1:5173/stats
```

5. 演示路径：

- 从第一章高精度进入一节课，展示视频、讲义、示例代码、总结测验和练习入口。
- 在课程页右侧展示第一章掌握清单和回刷题包，说明学生如何完成课后收束。
- 点击“标记完成”，展示课时状态、侧边栏总进度和章节进度条同步变化。
- 进入一道练习，展示 Monaco C++ 编辑器、提交按钮、分层提示、题解要点、测试点反馈和最近提交。
- 提交 Accepted 后回到课程页，展示题目“已通过”和继续学习入口。
- 切到第三章递推，展示 `路径计数` 到 `数塔递推` 的递进关系。
- 打开 `/stats`，展示演示数据形成的通过率、活跃题目、错误状态分布，以及教师视角的薄弱题排行、学生练习观察和下一课建议。

6. 演示前检查：

```powershell
powershell.exe -ExecutionPolicy Bypass -File scripts/check_demo.ps1
cd frontend
npm.cmd run build
```

## 下一轮实现优先级

### P0：演示稳定性

- 用 `scripts/check_demo.ps1` 作为演示前静态与数据验收入口。
- 在 `frontend/` 目录直接运行 `npm.cmd run build` 做前端构建验收。
- 手动验收 `/stats` 在真实浏览器中的视觉表现。
- `demo_start.ps1` 和 `demo_stop.ps1` 已形成基础启停链路；后续可补更精确的 PID 文件和日志目录。

### P1：学生学习进度

目标：让 demo 不只是“能提交”，还要显示“学到哪里了”。

第一版已用浏览器本地状态实现，后续再迁移到后端用户体系：

- 课时完成状态：点击完成后标记，可再次取消。
- 题目通过状态：提交 Accepted 或读取到本题 Accepted 历史后自动点亮。
- 章节进度：已完成课时数、已通过题目数、继续学习入口。
- 学习路径提示：当前课时、下一课时、推荐复习题。

后续增强：

- 视频播放到接近结尾时自动提示标记完成。
- 增加“重置 demo 进度”按钮。
- 将本地进度同步到后端学生账号。

### P2：教师视角

目标：让统计看板从“全局提交统计”升级到“教学反馈看板”。

第一版已用当前 OJ 提交聚合实现，演示学生优先读取 `seed_demo_data.py` 写入的 `demo-seed` 标记：

- 班级概览：练习覆盖、需讲解题、平均尝试次数和高频错误。
- 薄弱题排行：按低通过率、无 Accepted、错误提交密度计算关注度。
- 错误类型分布：把 Accepted、Wrong Answer、Compile Error 等状态转换为可视化条。
- 学生练习观察：按演示学生聚合提交数、通过题、通过率、关注点和最近活跃时间。
- 下一次课建议：根据薄弱题、错误类型和需要帮扶的学生生成教学动作。

后续建议后端逐步增加：

- `students`：学生基础信息。
- `classes`：班级信息。
- `student_id`、`class_id`：提交记录归属。
- 班级维度统计：班级通过率、活跃学生、薄弱题目。
- 学生维度统计：个人最近提交、通过题目、常见错误。

前端后续增强：

- 班级筛选、日期范围筛选和章节筛选。
- 真实学生列表、个人学习报告和最近活跃时间线。
- 教师对题目打标签、写讲评备注和生成课后复习包。

### P3：分层提示与题解

目标：摆脱“刷题式卡住”的体验。

第一版已覆盖第一章高精度：`frontend/src/data/support.ts` 为所有高精度练习题提供分层提示、题解要点和常见坑，题目页右侧默认展示第一层提示，并允许逐条展开。

每道题逐步补充：

- 第一层提示：只提醒思路方向。
- 第二层提示：指出关键变量或循环边界。
- 第三层提示：给出核心代码片段。
- 完整题解：解释状态定义、转移过程、边界和复杂度。
- 常见错误：越界、初始化遗漏、输出格式、下标偏移等。

### P4：章节总结测验

目标：让章节形成收束，而不是只是一串课时。

第一版已覆盖第一章高精度：课程页右侧展示 6 题即时反馈小测、掌握清单、章节进度和回刷题包。

后续可继续补：

- 更多面向不同学生水平的章节测验变体。
- 章节学习报告。
- 推荐复习课时。
- 教师端一键生成课后复习包。
