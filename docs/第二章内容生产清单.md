## 推荐续作顺序

1. `C02-01`：先补章节导入，让学生知道为什么学排序。
2. `C02-03`：从已有冒泡课时自然延伸到提前结束优化。
3. `C02-09`：用同分学生卡解释稳定性，连接冒泡、插入、选择三种算法。
4. `C02-10`：把六种算法放到同一张选择地图中比较。
5. `C02-11`：进入 C++ `struct` 与比较器的真实使用场景。
6. `C02-12`：以第 k 小收尾，建立“不必完整排序”的算法意识。

## 每支新视频的固定流程

1. 先在 `docs/视频脚本/` 写课时定位和分镜安排。
2. 再写逐镜旁白、画面动作、代码高亮与练习衔接。
3. 完成 Manim 场景后先低清渲染，检查文字、边界、节奏和颜色语义。
4. 低清验收后进入高清队列，并在课程数据中接入视频、封面和真实时长。
5. 运行 `scripts/check_demo.ps1` 与前端构建，确认媒体路径和题目 ID 一致。

## 验收口径

- 学生不看代码也能从动画说出本课的核心动作。
- 画面中的指针、边界、已排序区与代码变量含义一致。
- 同一位置不同时出现互相遮挡的标题、说明或指针标签。
- 最后一个镜头给出复杂度、适用场景和至少一个常见误区。
- 每支视频至少连接 2 道练习，课程页显示的时长与成片相符。
